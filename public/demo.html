x402 Payment Flow

How Mercury processes requests

1
Client sends GET request
Agent requests Treasury yield curve data
GET /v1/treasury/yield-curve/daily-snapshot
↓
2
Server returns 402 Payment Required
Payment terms included in response
HTTP/2 402
amount: "10000" // $0.01 USDC
payTo: "0xf8d5...81c6d2"
↓
3
Client signs USDC authorization
EIP-3009 signature on Base mainnet
network: "eip155:8453" // Base
asset: "USDC"
signature: "0x..."
↓
4
Client replays request with proof
Same endpoint, now with payment signature
GET /v1/treasury/yield-curve/daily-snapshot
payment-signature: "0x..."
↓
5
Server returns 200 OK + data
Deterministic JSON snapshot
HTTP/2 200
{ "curve_date": "2026-02-10",
  "rates": {...} }

One request. One microtransaction. Immutable data.
